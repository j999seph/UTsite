# OOO - Order Of Operations

## Purpose
This file is the working protocol for UTsite. Keep all project decisions, notes, and evolving requirements here so we can reference and revise them later without losing context.

## Operating rules
- Capture every meaningful user request or change request for this project.
- Keep the current decision set visible and update it as feedback changes.
- Treat unresolved items as active until the user confirms they are satisfied, or explicitly asks to revisit later.
- Prefer minimal, quiet, premium design decisions unless the user changes direction.
- Preserve the single-page, anchored-scroll structure.
- Keep the site discreet, capable, globally connected, and not overly explanatory.
- Maintain a running log of new asks, decisions, and open questions in the notes folder.
- If a requirement is delayed pending client feedback, mark it as pending rather than dropping it.

## Current stack direction
- Frontend: Next.js
- Styling: Tailwind CSS
- Animations: Framer Motion
- Hosting: Vercel
- Forms: Formspree, Resend, or custom API
- CMS optional: Sanity, Contentful, or Strapi

## Content and UX direction
- One long page with anchored navigation
- Minimalist information density
- Strong typography and premium spacing
- Calm, dark theme
- Smooth scrolling and subtle motion only
- Mobile-first clarity
- Quietly selective tone that discourages low-quality inquiries
- The first pass should prioritize clarity and positioning over feature depth.
- Contact should support a simple inquiry path plus room for future qualification fields.

## Open questions
- Final contact method and form destination
- Final image direction and whether we use generated visuals, sourced visuals, or both
- Whether a future CMS is needed at all
- Final hosting and form provider choice after the client responds
- Whether to keep the current custom API endpoint or swap to a third-party form provider
